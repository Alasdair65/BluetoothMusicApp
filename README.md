# 🎵 蓝牙音乐自动化 (BluetoothMusicApp)

一款专业的安卓应用，当连接到指定蓝牙设备时**自动播放 Apple Music**，支持 **BGM 歌曲 → 播放列表** 自动切换功能。

> 🎯 **核心功能**：连接蓝牙 → 播放指定 BGM 歌曲 → BGM 播放完成 → 自动切换到播放列表

## ✨ 功能特性

### 🔗 蓝牙触发
- 自动检测指定蓝牙设备连接
- 支持蓝牙耳机、车载蓝牙、音箱等
- 5 分钟冷却时间，避免重复触发
- 后台持续监控，低功耗运行

### 🎼 Apple Music 完整支持
| 功能 | 支持状态 | 说明 |
|------|----------|------|
| 启动应用 | ✅ | 自动打开 Apple Music |
| 播放 BGM 歌曲 | ✅ | 搜索并播放指定歌曲 |
| BGM 完成检测 | ✅ | 自动检测歌曲播放完成 |
| 切换到播放列表 | ✅ | BGM 完成后自动播放歌单 |
| 播放模式设置 | ✅ | 随机/顺序/单曲循环 |
| 无障碍控制 | ✅ | 精确 UI 操作支持 |

### 🎹 BGM → 播放列表 自动切换
```
连接蓝牙
    ↓
启动 Apple Music
    ↓
播放 BGM 歌曲（例如："Welcome Home"）
    ↓
检测 BGM 播放完成
    ↓
自动切换到播放列表（例如："我的最爱"）
    ↓
按设定模式播放（随机/顺序/循环）
```

## 🛠️ 构建说明

### 方法一：Android Studio（推荐）

1. **打开项目**
   ```
   启动 Android Studio → File → Open → 选择 BluetoothMusicApp 文件夹
   ```

2. **同步 Gradle**
   - 打开后 Android Studio 会自动同步 Gradle 依赖
   - 等待同步完成

3. **配置 SDK 路径**
   - 编辑 `local.properties` 文件
   - 修改 `sdk.dir` 为你的 Android SDK 路径
   
   **Windows:**
   ```
   sdk.dir=C:\\Users\\你的用户名\\AppData\\Local\\Android\\Sdk
   ```
   
   **macOS:**
   ```
   sdk.dir=/Users/你的用户名/Library/Android/sdk
   ```
   
   **Linux:**
   ```
   sdk.dir=/home/你的用户名/Android/Sdk
   ```

4. **构建 APK**
   ```
   Build → Build Bundle(s) / APK(s) → Build APK(s)
   ```

5. **安装到手机**
   - 通过 USB 连接手机
   - 运行 `Run → Run 'app'`
   - 或手动安装生成的 APK 文件

### 方法二：命令行构建

```bash
# 进入项目目录
cd BluetoothMusicApp

# 赋予 Gradle 执行权限（Linux/macOS）
chmod +x gradlew

# 构建 Debug 版本
./gradlew assembleDebug

# 构建 Release 版本（需要先配置签名）
./gradlew assembleRelease

# APK 输出位置
# Debug: app/build/outputs/apk/debug/app-debug.apk
# Release: app/build/outputs/apk/release/app-release-unsigned.apk
```

## 📱 使用指南

### 首次设置

#### 1. 安装应用
- 将 APK 文件传输到手机
- 点击安装（可能需要允许"未知来源"）

#### 2. 授予权限
打开应用后，需要授予以下权限：

| 权限 | 用途 | 是否必需 |
|------|------|----------|
| 蓝牙连接 | 检测蓝牙设备连接 | ✅ 必需 |
| 蓝牙扫描 | 扫描已配对设备 | ✅ 必需 |
| 通知 | 显示后台服务状态 | ✅ 必需（Android 13+） |
| 无障碍 | 控制 Apple Music 界面 | ⚠️ 推荐（完整功能需要） |

#### 3. 开启无障碍服务（重要！）

为了完整支持 **BGM→播放列表** 自动切换，需要开启无障碍服务：

1. 首次保存配置时，应用会提示开启无障碍权限
2. 点击"去设置"打开无障碍设置页面
3. 找到"蓝牙音乐控制"服务
4. 开启开关

**无障碍服务的作用：**
- 在 Apple Music 中搜索指定歌曲
- 点击播放按钮
- 检测 BGM 播放完成
- 自动切换到播放列表
- 设置播放模式（随机/顺序/循环）

> 🔒 **隐私说明**：此服务仅在 Apple Music 应用运行时激活，不会收集任何个人信息。

#### 4. 配置自动化

**步骤 1：选择蓝牙设备**
- 从列表中选择你的蓝牙设备（耳机/车载蓝牙/音箱）
- 如果列表为空，请先在系统设置中配对设备

**步骤 2：选择音乐应用**
- 选择 **Apple Music**（推荐，完整功能支持）
- 或选择其他音乐应用

**步骤 3：设置 BGM 歌曲**
- 输入连接蓝牙后想先播放的歌曲名
- 例如："Welcome Home - Mr. Mister"
- **留空则直接播放歌单，不播放 BGM**

**步骤 4：输入播放列表名称**
- 输入 BGM 播放完成后要播放的歌单名称
- 例如："我的最爱"、"驾驶歌单"、"晨跑音乐"
- **请确保在 Apple Music 中已创建此歌单**

**步骤 5：选择播放模式**
- 顺序播放：按歌单顺序依次播放
- 随机播放：随机播放歌单中的歌曲
- 单曲循环：重复播放当前歌曲

#### 5. 保存并启动
- 点击"保存配置并启动监控"
- 应用会在后台运行，通知栏显示监控状态

### 播放流程状态机

```
IDLE (空闲)
    │
    │ 检测到蓝牙连接
    ▼
LAUNCH_APP (启动应用)
    │
    │ 延迟 3 秒
    ▼
PLAY_BGM (播放 BGM)
    │
    │ 发送播放指令
    ▼
WAIT_BGM (等待 BGM 完成)
    │
    │ 检测播放状态变为暂停/停止
    ▼
PLAY_PLAYLIST (播放播放列表)
    │
    │ 设置播放模式
    ▼
COMPLETE (完成)
    │
    │ 10 分钟后
    ▼
IDLE (重置为空闲)
```

## ⚠️ 重要注意事项

### Apple Music 使用要求

1. **必须安装 Apple Music 应用**
   - 从 Google Play 或官网下载
   - 需要有效的 Apple Music 订阅

2. **必须提前创建播放列表**
   - 在 Apple Music 中创建歌单
   - 确保歌单名称与配置中完全一致
   - 建议歌单至少有 3-5 首歌曲

3. **首次使用需要手动登录**
   - 打开 Apple Music 并登录
   - 确保可以正常播放音乐

### 无障碍服务说明

**为什么需要无障碍权限？**

Apple Music 安卓版没有公开的 API 来直接播放特定歌曲。为了实现 **BGM→播放列表** 的完整功能，我们使用无障碍服务来：

1. 在 Apple Music 界面中搜索歌曲
2. 点击播放按钮
3. 检测当前播放状态
4. 在 BGM 完成后点击播放列表

**隐私安全：**
- 服务仅在 Apple Music 应用运行时激活
- 不会收集或上传任何个人信息
- 不会访问其他应用的数据
- 源代码完全开源可审查

### 电池优化

为确保后台服务正常运行：

1. 进入系统设置 → 应用 → 蓝牙音乐自动化
2. 电池 → 关闭"电池优化"
3. 允许"后台活动"

### 自启动权限

部分安卓手机（小米、华为、OPPO、vivo 等）需要额外设置：

1. 手机管家 → 应用启动管理
2. 找到"蓝牙音乐自动化"
3. 允许"自启动"和"后台运行"
4. 锁定应用到最近任务（防止被清理）

## 🐛 故障排查

### 问题 1：应用无法检测蓝牙设备

**可能原因：**
- 蓝牙未启用
- 设备未配对
- 权限未授予

**解决方案：**
1. 检查蓝牙是否已启用（下拉通知栏）
2. 在系统设置 → 蓝牙中配对设备
3. 重新授予蓝牙权限（设置 → 应用 → 蓝牙音乐自动化 → 权限）
4. 重启应用

### 问题 2：音乐不自动播放

**可能原因：**
- Apple Music 未安装
- 无障碍服务未开启
- 媒体权限未授予

**解决方案：**
1. 确认 Apple Music 已安装并登录
2. 开启无障碍服务（设置 → 无障碍 → 蓝牙音乐控制）
3. 检查媒体权限（设置 → 应用 → 蓝牙音乐自动化 → 权限）
4. 手动打开一次 Apple Music 并播放歌曲

### 问题 3：BGM 播放完成后不切换到播放列表

**可能原因：**
- 无障碍服务未运行
- 播放列表名称不匹配
- 播放状态检测失败

**解决方案：**
1. 检查无障碍服务是否开启
2. 确认播放列表名称与 Apple Music 中完全一致
3. 查看应用日志（设置 → 开发者选项 → 记录日志）
4. 尝试重启应用和手机

### 问题 4：应用被系统杀死

**解决方案：**
1. 关闭电池优化（见上方）
2. 锁定应用到最近任务
3. 允许自启动权限
4. 在通知栏点击"正在监控"通知，防止被清理

### 问题 5：找不到播放列表

**解决方案：**
1. 在 Apple Music 中确认歌单已创建
2. 检查歌单名称是否包含特殊字符
3. 尝试使用更简单的歌单名称
4. 确保歌单不是空的

## 🔧 高级配置

### 调整检查间隔

编辑 `BluetoothMonitorService.kt`：
```kotlin
private const val CHECK_INTERVAL_MS = 5000L // 改为其他值（毫秒）
```

### 调整冷却时间

编辑 `BluetoothMonitorService.kt`：
```kotlin
if (now - lastConnectedTime > 300000) // 改为其他值（毫秒）
```

### 禁用 BGM 功能

在配置中留空 BGM 歌曲名称，应用会直接播放播放列表。

## 📦 发布签名

### 生成签名密钥

```bash
keytool -genkey -v -keystore bluetooth-music.keystore \
  -alias bluetooth-music -keyalg RSA -keysize 2048 \
  -validity 10000
```

### 配置签名

在 `app/build.gradle.kts` 的 `android` 块中添加：

```kotlin
signingConfigs {
    create("release") {
        storeFile = file("../bluetooth-music.keystore")
        storePassword = "你的密钥库密码"
        keyAlias = "bluetooth-music"
        keyPassword = "你的密钥密码"
    }
}

buildTypes {
    release {
        signingConfig = signingConfigs.getByName("release")
        // ... 其他配置
    }
}
```
