## 🎯 核心功能实现说明

### 1. BGM → 播放列表 自动切换流程

```
用户配置
   │
   ├─ BGM 歌曲：「Welcome Home」
   └─ 播放列表：「我的最爱」
   │
   ▼
蓝牙连接检测
   │
   ▼
启动 Apple Music
   │
   ▼
播放 BGM 歌曲 ─────────────┐
   │                       │
   ▼                       │
监控播放状态               │
   │                       │
   ├─ 正在播放 ────────────┘
   │
   └─ 播放完成（PAUSED/STOPPED）
         │
         ▼
   自动切换到播放列表
         │
         ▼
   设置播放模式
         │
         ▼
   完成（10 分钟后可再次触发）
```

### 2. 三种控制方式组合

| 方式 | 用途 | 可靠性 |
|------|------|--------|
| **Intent URL** | 打开搜索页面 | ⭐⭐⭐⭐ |
| **MediaController** | 播放/暂停控制 | ⭐⭐⭐⭐⭐ |
| **无障碍服务** | UI 模拟点击 | ⭐⭐⭐⭐ |

### 3. 状态管理

```kotlin
enum class PlaybackStep {
    IDLE,           // 空闲
    LAUNCH_APP,     // 启动应用
    PLAY_BGM,       // 播放 BGM
    WAIT_BGM,       // 等待 BGM 完成
```

## ⚠️ 重要提示

### 1. 无障碍权限必须开启

这是实现 **BGM→播放列表自动切换** 的关键。

**设置路径**：
```
设置 → 无障碍 → 已下载的服务 → 蓝牙音乐控制
```

详见：`无障碍权限设置指南.md`

### 2. 关闭电池优化

确保后台服务不被系统杀死：

```
设置 → 应用 → 蓝牙音乐自动化 → 电池 → 允许后台活动
```

### 3. Apple Music 限制

Apple Music 安卓版没有公开的 Intent API，我们使用了组合方案：
- 搜索 Intent 打开页面
- MediaController 控制播放
- 无障碍服务模拟 UI 操作

详见：`Apple_Music 完整支持说明.md`
